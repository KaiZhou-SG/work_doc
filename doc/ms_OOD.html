Object Oriented Design of the Messaging System Simulator

This document describes the clue of how is the Messaging System Simulator designed.

Basic Principles

The design should be flexible so that:

  1. Existing messages are easy to be amended, including adding or removing fields.
  2. New messages are easy to be included into the system.
  3. Messages and their responses are logged and retrievable.
  4. Self aware so that it can perform error checking and log errors.

Detailed rules

  1. Composition over inheritance, so that different type of messages don't affect each other.
  2. A message is divided into head and body.
  3. The type of a message can be uniquely identified by type_code.
  4. Each message has own method to perform self-check.
