The infrastructure of the simulator

Front ends and back ends

The simulator has both front ends and back ends

The fronts ends are:

  1. The Transport Planning Module(TM)
  2. The Driver MDI(called MDI)
  3. The Growth Venture Messaging Forwarder(Forwarder)

The back ends are web services:

  1. The web service hosted by Innosys Pte Ltd(called WS1)
  2. The web service hosted by Growth Venture Pte Ltd(called WS2)

How to deploy the above components

  1. The two back ends will be installed to two different pcs with static ip and IIS
  2. The TM will be installed on the pc which hosts WS1
  3. The MDI will be installed on the pc which hosts WS2
  4. The Forwarder will be installed on the pc which hosts WS2

How these components cooperate

  1. TM send a message to WS1
  2. WS1 processes and forwards the message to WS2
  3. WS2 then sends the message to the Forwarder
  4. The forwarder processes the message and send it to MDI
  5. The MDI responses to the message by sending another message to WS1
  6. WS1 responses to the message from MDI by making changes on TM schedule control

P.S.
No acknowledgement messages are discussed so far.
And to be compatible to what USS can provide, WS1 and WS2 will be installed into one pc.
